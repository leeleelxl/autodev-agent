用户要求创建流程化的记忆审查机制，确保重开对话时能立即恢复所有历史信息。

**Why:** 防止因上下文过长导致信息丢失，保持项目开发的连续性。

**How to apply:**

## 每次对话开始时（必须执行）

1. **立即读取** `.claude/memory/MEMORY.md` 索引
2. **加载所有** 索引中列出的记忆文件
3. **理解上下文** 基于记忆快速恢复项目状态
4. **向用户确认** 简要说明已加载的记忆内容

## 对话过程中

- 做出重要决策后 → 立即保存到 memory
- 完成功能模块后 → 更新项目进度
- 遇到问题并解决 → 记录经验教训
- 用户提出新要求 → 更新用户偏好

## 对话结束前

- 主动询问是否需要保存新信息
- 使用 `/sync-memory` 进行全面审查
- 确保所有重要信息已持久化

## 记忆文件规范

- 使用标准 frontmatter（name, description, type）
- 按类型分类（user/feedback/project/reference）
- 保持 MEMORY.md 索引更新
- 文件名清晰描述内容
