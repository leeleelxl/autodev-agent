# AutoDev Web UI

## 启动方式

### 完整版界面

```bash
streamlit run app.py
```

功能：
- 任务执行和结果展示
- 记忆浏览和搜索
- 工作流可视化
- 性能分析和统计

### 简化版界面

```bash
streamlit run app_simple.py
```

功能：
- 快速任务执行
- 代码结果展示
- 轻量级界面

## 界面功能

### 1. 任务执行 Tab

- **任务输入**: 文本框输入开发需求
- **配置选项**:
  - LLM 提供商选择（Claude/Kimi/Qwen）
  - API Key 配置
  - Docker 沙箱开关
  - 记忆系统开关
- **执行按钮**: 一键启动工作流
- **实时进度**: 显示当前执行阶段
- **结果展示**:
  - 执行状态（成功/失败）
  - 各阶段详情
  - 生成的代码文件
  - 测试用例
  - 审查结果

### 2. 记忆浏览 Tab

- **搜索功能**: 关键词搜索历史记忆
- **记忆列表**: 显示搜索结果
- **统计信息**:
  - 短期记忆数量
  - 长期记忆数量
- **最近记忆**: 显示最近的 5 条记忆

### 3. 工作流可视化 Tab

- **执行时间线**: 进度条显示各阶段
- **流程图**: Mermaid 图展示工作流
- **阶段详情**: 每个阶段的执行情况

### 4. 性能分析 Tab

- **Token 使用统计**: 各阶段 token 消耗
- **代码质量仪表盘**: 0-100 分可视化
- **问题分布**: High/Medium/Low 问题统计
- **性能指标**: 执行时间、成功率等

## 配置说明

### API Key 配置

在侧边栏输入对应 LLM 的 API Key：

- **Claude**: https://console.anthropic.com/
- **Kimi**: https://platform.moonshot.cn/
- **Qwen**: https://dashscope.aliyun.com/

### Docker 模式

- **开启**: 在隔离的 Docker 容器中执行代码（推荐）
- **关闭**: 在本地直接执行（开发测试）

### 记忆系统

- **开启**: 使用历史经验增强决策
- **关闭**: 不使用记忆（首次运行或测试）

## 使用流程

1. **配置 API Key**: 在侧边栏选择 LLM 并输入 Key
2. **输入任务**: 在文本框描述开发需求
3. **点击执行**: 启动工作流
4. **查看结果**:
   - 实时查看各阶段进度
   - 查看生成的代码
   - 查看测试结果
   - 查看审查报告
5. **浏览记忆**: 切换到记忆 Tab 查看历史经验

## 依赖安装

```bash
pip install streamlit plotly
```

## 端口配置

默认端口: 8501

自定义端口:
```bash
streamlit run app.py --server.port 8080
```

## 开发模式

```bash
streamlit run app.py --server.runOnSave true
```

自动重载，方便开发调试。
